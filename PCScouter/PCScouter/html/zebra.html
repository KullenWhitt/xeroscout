The XeroScout scouting software provides for some new features to analyze zebra data from the Blue Alliance. Zebra data is recorded during the match and provides a x/y location for reach robot in a given match at about 100 ms intervals. This data can be analyzed to deduce information about the behaviour of a given robot. In generaly the Zebra data support in the scouting program consists of:
  • Visualization of robot position data for all robots in a match
  • Visualization of robot position data for a robot across all matches
  • Visualization of a position heatmap for all robots in a match
  • Visualization of a position heatmap for a robot across all matches
  • Visual playback of the movement for all robots in a match
  • Visual playback of the movement for a robot across all matches
  • Analysis of rebot movement patterns


The analysis of robot movement patterns is experimental and consists of the following steps:
  • Define specific regions on the field that are interesting. For instance, the control panel on each side of the field could be a region. A region can be active for both alliances, or can be alliance specific.
  • Analyze the zebra data and determine when a robot enters and exits a given region. These are stored as robot events.
  • Analyze the zebra data and determine when the robot is sitting idle. These are stored as robot events.
  • Filter the event sequence to reduce the noise. This means doing things like taking an event sequence like ENTER->EXIT->ENTER->EXIT that occurs within a single second and reducing it to a single ENTER->EXIT event sequence.
  • Apply pattern matching to the event sequence to create a set of robot activities. For instance, a sequence like ...
    • Enter Loading Station Event
    • Optional Idle Event
    • Exit Loading Station Event
    • Enter Control Panel Event
    • Exit Control Panel Event
    • Enter Tranch Event
    • Required Idle Event
